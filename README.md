# Dark Luxury Editorial Web Skill

不是把攻略贴进网页。  
是把一份旅行材料，做成一个真的值得分享的旅行小站。

这个 skill 专门用来处理那种“内容明明很好，但网页一做出来就气质全失”的场景。  
它管的不只是文案怎么摆，更会盯首屏、次页、图片、标签、音乐、移动端和构建体积这些最容易翻车的地方。

## 它解决什么问题？

很多旅行网页并不是“内容不够”，而是败在这几件事上：

- 首屏有图，但首屏和次页像两块硬拼的板子
- AI 会做页面，却很容易做成“黑底加几段字”
- 标签乱打、配图跑偏、时间轴生硬，越做越像模板
- 移动端一看就露馅，字太细、层级太乱、视觉重心不对
- 背景音乐、素材 sourcing、字体体积这些细节总在最后一刻出问题

这个 skill 的意义就是：  
把这些坑提前变成规则，而不是等网页做完之后再一处一处返工。

## 哪些场景特别适合用它？

### 1. 你已经有一份现成路书，想把它变成网页

比如你手里已经有：

- 飞书文档
- Notion 页面
- Word / DOCX
- 聊天记录整理出来的行程

你不想只是“转网页”，你想要的是：

- 有封面感的首页
- 和次页自然融合的过渡
- 每天路线清楚，但不死板
- 发出去别人会觉得“这网页挺高级”

### 2. 你只有一个旅行意图，还没有成型攻略

比如你只知道：

- 从哪里去哪里
- 几天几夜
- 想要什么风格
- 大概预算

这个 skill 可以先把它整理成完整路书，再把路书做成网页。  
也就是说，它既能做“规划”，也能做“呈现”。

### 3. 你有一篇游记，想保留自己的文风

很多工具会把第一人称游记改成没有温度的总结。  
这套 skill 会明确区分“路书模式”和“游记模式”，尽量保留原来的口吻，只帮你把结构理顺、节奏拉出来。

### 4. 你已经有个半成品网页，但总觉得差一口气

特别适合拿来修这些老毛病：

- 首屏和次页割裂
- 时间轴做得太卡、太重、太花
- 标题不居中，视觉重心不稳
- 标签和配图语义不对
- 音乐播放器看起来像外挂组件
- 字体一加就把包体积炸大

## 它具体会怎么帮你？

### 第一步：先判断你给的是哪种输入

它会区分：

- 旅行意图
- 路书
- 第一人称游记

因为这三种材料，根本不该用同一种网页结构硬套。

### 第二步：先整理内容，再做视觉

它会先拆出：

- Hero 标题和副标题
- 每日结构
- 时间轴节点
- 地点 / 食物标签
- 行前准备、交通、住宿、美食、预算这些信息模块

先把结构理对，再进入视觉层。

### 第三步：先做图片和氛围，再做细节

这套 skill 明确要求：

- 先写 shot list
- 先找图
- 先把图落到本地
- 再做 Hero 和次页的融合

而不是最后才想起来“这里是不是应该有张图”。

### 第四步：把最难看的细节盯住

它会特别关注：

- Hero 和次页之间有没有明显切线
- 时间轴的线有没有穿透节点
- 标签是不是具体地点 / 食物，而不是空词
- 音乐播放器状态是否和真实播放一致
- 移动端字体、行高、重心、间距是否成立
- 中文字体有没有把构建体积拖爆

## 为什么它比普通网页生成更靠谱？

因为它不是只会说“做一个高级页面”。  
它已经把过去真实项目里反复踩过的坑，变成了具体的执行规则。

它特别适合这种目标：

- 我想做一个旅行网页，不想要 AI 味太重
- 我想保留审美，不想每次都从头调首屏过渡
- 我想让它既能做路书，也能做游记
- 我想让一个 skill 真的能从 0 到 1 跑出来，而不是只会生成一堆说明

## 仓库里有什么？

- `dark-luxury-editorial-web-skill/`
  完整 skill 源目录

- `dark-luxury-editorial-web-skill/SKILL.md`
  核心工作流、硬规则、视觉系统与 QA 规范

- `dark-luxury-editorial-web-skill/references/`
  benchmark、行程规划、文风约束、实现配方、媒体素材工作流

- `dist/dark-luxury-editorial-web-skill.skill`
  打包好的可安装 `.skill` 文件

## 你可以怎么用？

最常见的方式有三种：

1. 直接拿 `.skill` 文件安装，用它去跑新的旅行项目
2. 把仓库里的 `SKILL.md + references/` 当作可读可改的工作流基线
3. 拿它去校准你自己的 travel-web agent，让它别再把路书做成模板站
