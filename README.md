# Dark Luxury Editorial Web Skill

把飞书、Notion、Word 里的旅行材料，变成一个真的会让人想收藏、想转发、想问你“这个站怎么做的？”的旅行网页。

它不是把攻略“贴”到网页里。  
它更像一个很懂审美、也很懂前端落地的旅行网页制作人：会先帮你把封面图、次页过渡、时间轴、标签、配图、音乐和移动端排版都理顺，再把它们做成一个完整的小站。

## 它解决什么问题？

如果你做过旅行网页，你大概率经历过这种崩溃：

- 文案明明不错，做出来却像“黑底 + 几段字”
- 首屏看着还行，一往下滑就出现明显断层
- 图也有、动效也有，但就是不高级，甚至越做越像模板
- 手机一打开就露馅：字太细、层级太乱、重心不稳
- 临上线才发现音乐播不动、字体太重、图片不够、素材来源混乱

这个 skill 的作用，就是把这些最容易返工的地方，提前写成规则和 SOP。  
这样你不是在“事后救火”，而是在一开始就沿着正确的节奏把页面做出来。

## 产品亮点

- 傻瓜式：只需给出旅行需求，就能一站式生成完整路书和可发布网页
- 美观：高级杂志风格，几乎没有 AI UI 味道，自带氛围型背景音乐

## 适用人群

- 家庭旅行计划：想省心但又想保持质感的家庭出行
- 个体旅行社或小型工作室：用网页展示路线和体验亮点

## 哪些场景特别适合用它？

### 1. 你已经有攻略了，只差一个体面的呈现

比如你手里已经有：

- 飞书文档
- Notion 页面
- Word / DOCX
- 聊天记录整理出来的行程

这时候你需要的，不是“导出成网页”。  
你真正想要的是：

- 有封面感的首页
- 和次页自然融合的过渡
- 每天路线清楚，但不死板、不像旅游模板
- 发出去别人会觉得“你这个做得也太精致了吧”

### 2. 你只有一个旅行意图，还没有成型路书

比如你只知道：

- 从哪里去哪里
- 几天几夜
- 想要什么风格
- 大概预算

这时候它不只是网页生成器，也是一套从 0 到 1 的路书工作流。  
它会先把路线、节奏、模块和信息结构补完整，再把它做成可交互网页。

### 3. 你有一篇游记，不想被 AI 改得没了语气

很多工具会把第一人称游记改成一篇“信息没问题，但完全不像你写的”总结。  
这套 skill 会明确区分“路书模式”和“游记模式”，尽量保留原来的口吻和情绪，只帮你把结构和节奏理顺。

### 4. 你已经有个半成品网页，但总觉得差一口气

它也很适合专门拿来修这些顽固问题：

- 首屏和次页割裂
- 时间轴做得太卡、太重、太花
- 标题不居中，视觉重心不稳
- 标签和配图语义不对
- 音乐播放器看起来像外挂组件
- 字体一加就把包体积炸大

## 它具体会怎么帮你？

它的工作方式其实很简单：

1. 先判断你给的是旅行意图、路书，还是第一人称游记  
2. 先把结构理顺，再去做视觉：Hero、每日安排、时间节点、地点 / 食物标签、交通住宿、预算模块都会先归位  
3. 先写 shot list，再找图、落图、配氛围，避免页面做到最后才发现“好像少了图”  
4. 最后统一检查最影响质感的细节：首屏过渡、时间轴节点、标签语义、音乐状态、移动端层级、字体体积

## 你可以怎么用？

- 丢给它一份飞书 / Notion / Word / 聊天整理稿，让它直接做成旅行网页
- 只给“从哪里到哪里、几天几夜、什么气质、多少预算”，让它先补完整套路书再出网页
- 拿它去修你已经做了一半、但总不够高级的旅行站
- 把它当成你自己的 travel-web agent 基线，后面每个目的地都能复用

## 为什么它比普通网页生成更靠谱？

因为它不是只会说一句“做一个高级页面”。  
它把真实项目里反复踩过、反复返工的坑，全都写成了明确的执行规则。

你可以把它理解成：  
一个已经做过几轮实战、知道哪里最容易翻车的 travel-web 制作人，  
而不是一个随机吐一版模板出来的网页生成器。

## 仓库里有什么？

- `dark-luxury-editorial-web-skill/`
  完整 skill 源目录

- `dark-luxury-editorial-web-skill/SKILL.md`
  核心工作流、硬规则、视觉系统与 QA 规范

- `dark-luxury-editorial-web-skill/references/`
  benchmark、行程规划、文风约束、实现配方、媒体素材工作流

- `dist/dark-luxury-editorial-web-skill.skill`
  打包好的可安装 `.skill` 文件

## 最常见的使用方式

1. 直接安装 `.skill` 文件，用它跑新的旅行网页项目
2. 把仓库里的 `SKILL.md + references/` 当作你自己的 SOP 基线
3. 用它去校准你的 travel-web agent，让它别再把路书做成模板站
